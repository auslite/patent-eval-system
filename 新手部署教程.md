# 专利技术方案三方评估系统 - 新手部署教程（Bmob版）

## 📌 方案说明

本方案使用 **Bmob 后端云** 作为后端数据库：
- ✅ 国内访问稳定，无需翻墙
- ✅ 有免费版（开发版，免费3个月）
- ✅ 老牌BaaS服务，稳定可靠
- ✅ 支持多人实时协作
- ✅ 注册简单，新手友好

---

## 🚀 部署步骤（超详细版）

### 第一步：注册 Bmob 账号

1. 打开浏览器，访问：https://www.bmobapp.com
2. 点击右上角的「注册」按钮
3. 使用邮箱注册（建议使用常用邮箱）
4. 到邮箱查收验证邮件，点击链接完成验证
5. 登录账号

---

### 第二步：创建应用

1. 登录后，点击「我的控制台」
2. 点击「创建应用」按钮
3. 填写应用信息：
   - **应用名称**：`专利评估系统`（可以自定义）
   - **应用类型**：选择「普通应用」
4. 点击「创建」

---

### 第三步：获取应用密钥

1. 创建应用后，点击应用名称进入应用管理页面
2. 点击顶部菜单「设置」→「应用密钥」
3. 找到以下信息并**复制保存**下来：
   - **Application ID**：类似 `1234567890abcdef1234567890abcdef`
   - **REST API Key**：类似 `fedcba0987654321fedcba0987654321`

> ⚠️ **重要**：这些信息相当于你的数据库密码，不要泄露给他人！

---

### 第四步：创建数据表

1. 在应用管理页面，点击「数据」标签
2. 点击「添加表」按钮

**创建第一个表 - Project（项目表）：**
1. 表名填写：`Project`
2. 点击「创建」
3. 添加以下字段（点击「添加列」）：
   - `projectId`（String/字符串）- 项目唯一ID
   - `name`（String/字符串）- 项目名称
   - `description`（String/字符串）- 项目描述
   - `creator`（String/字符串）- 创建人
   - `images`（Array/数组）- 图片列表
   - `links`（Array/数组）- 链接列表
   - `created_at`（String/字符串）- 创建时间

**创建第二个表 - Evaluation（评估表）：**
1. 点击「添加表」按钮
2. 表名填写：`Evaluation`
3. 点击「创建」
4. 添加以下字段：
   - `evalId`（String/字符串）- 评估唯一ID
   - `projectId`（String/字符串）- 所属项目ID
   - `evaluatorId`（String/字符串）- 评估人ID
   - `evaluator`（String/字符串）- 评估人姓名
   - `department`（String/字符串）- 部门
   - `totalScore`（Number/数字）- 总评分
   - `subScores`（Object/对象）- 子评分
   - `submittedAt`（String/字符串）- 提交时间

> 💡 **简化操作**：其实不需要手动创建字段，系统会在第一次保存数据时自动创建！

---

### 第五步：设置权限（重要）

为了让多人可以协作编辑数据，需要设置权限：

1. 在「数据」页面，点击 `Project` 表
2. 点击「权限」标签
3. 将以下权限都设置为「所有人」：
   - 查询（find）
   - 插入（insert）
   - 修改（update）
   - 删除（delete）
4. 点击「保存」

对 `Evaluation` 表重复上述操作。

> ⚠️ **安全提示**：生产环境建议只给登录用户权限，这里为了简化先给所有人权限

---

### 第六步：修改配置文件

1. 下载我提供的代码文件（4个文件）
2. 用文本编辑器（如记事本、VS Code）打开 `config.js` 文件
3. 修改以下配置：

```javascript
const BMOB_CONFIG = {
    // 替换为你的 Application ID
    APPLICATION_ID: '你的实际ApplicationID',
    
    // 替换为你的 REST API Key
    REST_API_KEY: '你的实际RESTAPIKey',
    
    // 服务器地址（不用改）
    HOST: 'https://api.bmobapp.com',
    
    // 邀请码（可以修改为你喜欢的）
    INVITE_CODE: '5201314'
};
```

4. 保存文件

---

### 第七步：部署网站

#### 方案 A：Vercel（免费，推荐）

1. 访问 https://vercel.com
2. 点击「Sign Up」注册账号（可以用 GitHub 账号直接登录）
3. 登录后，点击「Add New...」→「Project」
4. 选择「Import Git Repository」或者直接用本地上传
5. 上传你的 4 个文件（index.html、app_logic.js、config.js、styles.css）
6. 点击「Deploy」
7. 等待部署完成（通常 1-2 分钟）
8. 部署完成后，会给你一个网址

#### 方案 B：腾讯云静态网站托管（国内访问更快）

1. 访问 https://console.cloud.tencent.com
2. 搜索「静态网站托管」或「云开发 CloudBase」
3. 创建环境（选择免费版）
4. 上传你的文件

#### 方案 C：Gitee Pages（国内访问快）

1. 访问 https://gitee.com
2. 注册账号
3. 创建仓库
4. 上传文件
5. 开启 Gitee Pages 服务

---

### 第八步：测试使用

1. 打开部署后的网址
2. 使用邀请码登录（默认是 `5201314`）
3. 创建一个测试项目
4. 退出登录，重新登录，检查项目是否还在
5. 让同事访问同一网址，测试多人协作

## 🔧 常见问题解决

### 问题 1：提示"云端连接失败"或"未配置"

**可能原因：**
- config.js 中的配置不正确
- Application ID 或 REST API Key 填写错误

**解决方法：**
1. 检查 config.js 中的配置是否正确
2. 确认 Application ID 和 REST API Key 是从 Bmob 控制台复制的
3. 按 F12 打开浏览器控制台，查看具体错误信息

### 问题 2：数据无法保存

**可能原因：**
- 权限未设置
- 网络问题

**解决方法：**
1. 检查 Bmob 中的权限设置
2. 刷新页面重试
3. 检查浏览器控制台错误信息

### 问题 3：同事看不到我创建的项目

**可能原因：**
- 数据保存在本地，未同步到云端
- 同事访问的是不同网址

**解决方法：**
1. 确保所有人都访问同一个网址
2. 检查 config.js 配置是否正确
3. 在 Bmob 控制台查看数据是否已保存
